# スキル作成スキル

## このスキルの目的
ユーザーへのヒアリングを通じて、統一フォーマットの SKILL.md ファイルを生成する。

## 作成手順

以下の手順でユーザーにヒアリングし、スキルを作成すること。

### Step 1: スキルの目的を聞く
ユーザーに「どんなスキルを作りたいか」「スキルで何をしたいか」を質問する。

### Step 2: スキル名を決める
ヒアリング内容をもとにケバブケースのスキル名を提案し、ユーザーに確認する。
- 例：`meeting-notes`、`code-review`、`api-design`

### Step 3: 出力フォーマットを確認する
スキルがファイルや特定の形式を出力する場合、その出力フォーマットをユーザーに確認する。
出力フォーマットが不要なスキルの場合、このステップはスキップしてよい。

### Step 4: ルール・制約を確認する
スキル実行時に守るべきルールや制約をユーザーに確認する。
- 例：「○○がない場合は『不明』と記載する」「話し言葉は書き言葉に直す」

### Step 5: 確認してから生成する
ヒアリング内容をまとめてユーザーに提示し、確認を取ってから SKILL.md ファイルを生成する。

## 出力フォーマット

以下の形式で `.claude/skills/{スキル名}/SKILL.md` にファイルを作成すること。

```markdown
---
name: {スキル名}
description: {スキルの説明（1行）}
---

# {スキルのタイトル}

## このスキルの目的
{スキルが何をするかの説明}

## 出力フォーマット
{出力がある場合、そのフォーマットを記述。出力がないスキルの場合はこのセクションを省略可}

## ルール
- {スキル実行時に守るべきルール}
```

## ルール
- ディレクトリ名はケバブケースにする
- フロントマターの `name` フィールドはディレクトリ名と一致させる
- フロントマターの `description` は1行で簡潔に記載する
- スキル本体は Claude への指示文として記載する（命令形で書く）
- SKILL.md を生成する前に、必ずユーザーに内容を確認してもらう
- 出力フォーマットのセクション内にテンプレートを記載する場合、フロントマターの区切り文字 `---` と衝突しないようコードブロックで囲む
