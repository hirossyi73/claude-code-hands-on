# 作業完了時の自発報告義務

本ファイルは、AI エージェントが作業を完了した際に、
**聞かれなくても自発的に気づきや懸念を報告する義務**を定義する。

---

## 1. 基本方針

- 作業完了時、成果物の報告に加えて、**作業中に気づいた懸念・疑問・改善案**を自発的に報告すること。
- ユーザーに「何か気になったことは？」と聞かれる前に、自分から報告する。
- 報告は **作業の最後に専用セクションとして** まとめて提示する。
- 気づきがゼロの場合は「特になし」と明示する（報告自体を省略しない）。

---

## 2. 報告すべき観点

以下の観点で作業中に気づいた点があれば報告する。

### 2.1 仕様・要件に関する気づき

- 仕様の曖昧さ・解釈が分かれそうな箇所
- 仕様と既存実装の矛盾
- 考慮されていない条件
- 「こう解釈して進めたが、合っているか確認したい」という判断

### 2.2 改善提案

- 「今回は考慮外だが、こうした方がよい」という改善案

---

## 3. 報告フォーマット

作業完了時に、以下のフォーマットで報告セクションを付与する。

```
## 作業中の気づき・懸念事項

### [観点カテゴリ]（例：仕様 / 設計 / 既存のファイルやコード / パフォーマンス / 改善提案）
- **対象**: [該当ファイル・箇所・機能名]
- **内容**: [気づいた内容]
- **影響度**: HIGH / MEDIUM / LOW
- **推奨アクション**: [対応案があれば記載]
```

### 3.1 影響度の基準

| 影響度 | 基準 |
|---|---|
| HIGH | 放置すると問題・バグ・障害・リスクにつながる可能性がある |
| MEDIUM | 品質や保守性に影響があるが、即座の対応は不要 |
| LOW | 改善すればベターだが、現状でも問題なく動作する |

---

## 4. 注意事項

- 報告は**事実と根拠に基づく**こと（推測の場合は「推測」と明記する）。
- 依頼範囲外の修正を報告に含める場合、**勝手に修正せず報告のみ**にとどめる。
- 報告の量が多い場合は、影響度 HIGH のものを優先して報告する。
- この報告義務は、調査・対応・質問など**すべての作業タイプに適用**される。
